# M4.1 — Feedback pass: the level signal + the sequence rework

> The plan of record for the batch of in-game feedback from the **first real
> play-test of the M4 build** (2026-07-22, CDMProbe v0.19.0). Short version:
> `milestones.md` M4.1. This doc is the *how*.
>
> **Doc map:** §0 direction → `spec.md` · §0.5 guidance → `guidance-model.md` ·
> milestone log → `milestones.md` · this is the M4.1 implementation plan (sibling
> of `m4-plan.md`).

## Context

M4 shipped code-complete and the log confirmed the engine works — strictness held
(`lit` mass at 1–2, peak 3), the capping rule fired ("build to cap for Tyrant"), the
stage-hold fired ("stage for Tyrant"), the recorder captured it all. **Playing it**
surfaced a different class of problem: the board is *correct* but doesn't *read*, the
chrome is heavier than it needs to be, and the sequence tool — the one instructional
surface — is structurally broken in two ways. The feedback splits into four
workstreams:

- **A — the icon-level signal doesn't read.** The dot is too subtle; `AVAILABLE`
  draws nothing, so a ready-but-"your call" **Implosion looks dead** even when it is
  the press; and the reason *row* is doing the glance-work the visual should. The
  player's proposal is a **right-side colour-graded bleed** ("a colored shadow") off
  the icon — more preattentive than a small dot, no line to read, and it lets the
  reason text drop out when `hud debug` is off.
- **B — chrome cleanup.** The shard rail is ~3 icons tall and too wide; the `C:\>_`
  terminal footer is noise; the `[.] PREP 3/5` mode label is unreadable and unneeded.
- **C — scoring / data.** A single wrong spellID (**Imp Lord `136726` → `1276452`**)
  is the root cause of two separate reports. Plus: Grimoire should stage for Tyrant,
  the opener's trailing SB rebuild step is proc-invalidated, and the fonts are small.
- **D — the sequence tool.** Two structural bugs (opener clobbered by BURST; burn
  pane dissolves at Tyrant cast) plus content gaps (no imp in the burst sequence; imp
  shown conditionally when its state is always knowable).

**The root-cause find (drives C and D):** `SpecDemonology` and `SpecOpener` reference
Imp Lord by `136726`, which is the **talent tree entry-id** (`talents.json:1917`), not
the **cast spellID** `1276452` (`talents.json:1919`, `talents.md:100`). The live
tracked button is `1276452` (probe-confirmed, both the OOC item list and the in-combat
override `1276452 → 132411` Singe Magic). Because `1276452` is unknown to `ns.Spec`, it
falls to the neutral/utility default → "your call" → **always AVAILABLE** (the "Imp
always available" report), and because no keybind resolves for `136726` the sequence
strip **prints the name "Imp Lord"** instead of the key (the "shows by name" report).
Fel Ravager `1276467` is already correct; only Imp Lord's number is wrong.

## Approach, per workstream

### A — the level signal becomes a right-side colour-graded bleed

Replace the masked-disc **dot** (`HudChrome.ensureDot`/`paintDot`/`SetDot`, and the
`DOT` table) with a **horizontal gradient texture** anchored to the icon's outer edge
(right for Essential, left for Utility, per `H.SideFor`), fading colour→transparent
outward (`Texture:SetGradient("HORIZONTAL", …)`). **Colour carries LEVEL** (unchanged
inversion from M3c-a); **spread + intensity carry it redundantly** so the signal is
never colour-alone ([X1]) and survives a dim monitor / colourblindness:

| Level | Hue | Spread | Motion | Reads as |
|---|---|---|---|---|
| `NEVER` | — | none | — | nothing (can't press) |
| `AVAILABLE` (held / overcap / "not calling it") | — | none | — | nothing (deliberately quiet) |
| **`AVAILABLE` + judge-ready** (judgeable=false and otherwise up) | cyan | narrow | steady | **ready — your call** (Implosion when up) |
| `SOON` (treatment on `NEVER`) | yellow | medium | gentle alpha pulse | coming up |
| `ROTATION` | green | wide | steady | press this now |
| `LATE` | green | wide | slow size/intensity pulse | overdue |

The new bit is **judge-ready**: `HudScore` sets a flag when `info.judgeable == false`
**and** the cooldown/gate is otherwise met (Implosion off cooldown), so the bleed can
light cyan "ready, your call" instead of drawing nothing. Plain `AVAILABLE` (utility
defensives, a held HoG, an overcapping generator) stays dark — we are *not* lighting
every castable button.

**Debug-off drops the reason text.** In non-verbose mode `HudRow` stops appending the
level word + reasons; the bleed is the whole signal. `hud debug` restores the full
reasoned rows (identity, cost, presence, override) unchanged — the correctness view.
The on-screen `lit` summary counter stays (it is the strictness meter, not per-row
noise). This **reverses M3c-a's "words-first default view"** — recorded as an
amendment in `guidance-model.md §0.5.8.7`, because the fix for illegible colour is a
legible *visual*, not permanent words.

### B — trim the chrome

- **Shard rail full-height + narrower.** `buildRail` sizes the rail from `RAIL_ICONS
  = 3` × pitch and `RAIL_SEG_W = 20`. Make it span the **whole viewer height** (measure
  the panel, not a fixed icon count) and narrow the width (~12). `iconMetrics` already
  measures pitch/height; the span becomes the viewer's height.
- **Drop the `C:\>_` footer** — delete the footer FontString + the `setBlink` ticker in
  `buildTerminal`/`ShowTerminal`/`HideTerminal`. Nothing else references it.
- **Drop the rail mode label** (`f.label`, "`[.] PREP 3/5`") — remove its creation and
  the `PaintRail` text build. Mode still reads via the rail-fill hue + the DEMO.SYS
  terminal tint (`SetTerminalMode`). ⚠ This gives up the [X1] redundant mode *label*;
  accepted because BURST — the one mode that matters — names itself on the sequence
  pane header, and the shard count still shows on the `HudRow` summary line.

### C — data + scoring + fonts

- **Imp Lord `136726` → `1276452`** in `SpecDemonology.ns.Spec` and `SpecOpener`/
  `SpecBurst` steps. Keep `136726` mapped as an alias (harmless, covers a build that
  surfaces it) but the primary is `1276452`.
- **`stage = true`** on both Grimoire summons (`1276452` Imp Lord, `1276467` Fel
  Ravager) so the BURST window reads them AVAILABLE "stage for Tyrant" like
  Dreadstalkers (`diabolist-sequences.md`: Imp Lord is a 2-min summon out **before**
  Tyrant). Keep them `optional` in the queue drain (2-min CD = present every *other*
  Tyrant; a missing one must drop-through, not jam).
- **Drop the SB×3 rebuild step** — remove `SpecOpener.steps[7]` (`SB · rebuild`). A
  scripted filler tail is thrown out by the first proc.
- **Fonts.** Standardise the sequence pane on `ns.SetFont` (JetBrains Mono — the font
  the `HudRow` reason rows use, which the player finds sharp) and bump sizes:
  `HudQueue` strip 12→~16 and header 9→~12; `HudPane` prereq row 10→~13. Leave the
  `HudRow` size (14) as the reference.

### D — the sequence rework

**D1 — BURST may not preempt an unfinished opener.** `HudBurst.OnMode`/`B.Arm` must
bail if the pane is currently owned by the opener: the opener's first Tyrant window
*is* the first burst window, and it dissolves on that window closing (its `tyrantAt`
clock) or on drain — after which subsequent BURST windows arm the burn pane normally.
One guard (`if ns.HudPane.OwnedBy("opener") then return end`) at the arm path.

**D2 — decouple the burn pane's lifecycle from the mode.** Today `B.OnMode(BURST)`
arms and `B.OnMode(≠BURST)` dissolves — but the mode leaves BURST the instant Tyrant
is cast (Tyrant → 60 s CD), so the burn drains one step and vanishes. The burn window
is the **~18 s after** the cast. Model:

- `B.OnMode(BURST)` — arm (in combat, enabled, opener not owning). This is the
  **windup**: primed on step 1, you stage Dreadstalkers / Imp Lord (their dots say
  "stage for Tyrant") and build to cap.
- `B.OnCast(TYRANT)` — mark `B.burning = true`, `B.tyrantCastAt = GetTime()`.
- `B.OnMode(≠BURST)` — **do not dissolve** while `burning` and within `BURN_WINDOW`;
  the mode correctly leaves BURST for SPEND (post-Tyrant you *dump*, so the capping
  rule releasing HoG is right) but the *pane* keeps walking HoG HoG → cores → Implosion.
- `B.Tick` — dissolve when the burn drains (`IsEmpty`), `BURN_WINDOW` (~18 s, Tyrant's
  active duration) elapses since the cast, or combat ends.

**D-content — the burst sequence gets its staging + imp.** `ns.SpecBurst.steps` today
is `[Tyrant, HoG×2, Demonbolt, Implosion]` — it starts at Tyrant, so it never mentions
the pre-Tyrant summons. Prepend the staged summons to mirror the opener and match
`diabolist-sequences.md`: `[Dreadstalkers, Imp Lord (opt), Tyrant, HoG×2, Demonbolt
(cores), Implosion (opt)]`. Add Imp Lord (`1276452`) and Dreadstalkers to
`SpecBurst.prereqs` so their readiness shows in the wall-down row.

**D-imp-known — drop the conditional framing.** With the correct ID the keybind
resolves and readiness is knowable (`readyAt` in combat, seed OOC), so the imp step
shows its key like every other step and its state shows in the prereqs row. Keep
`optional` for the drop-through (absence must not jam), but drop the "if up" *note*
that implied we couldn't tell.

## Implementation phases

**Phase A — the bleed + debug-off text (the headline).** Build the gradient-bleed
primitive in `HudChrome`, wire the judge-ready cyan, recompute `ROW_OFFSET`, and make
`HudRow` non-verbose text-free. *Exit:* at a dummy, 1–2 icons carry a clear coloured
bleed, Implosion lights cyan when up, and the board reads with `hud debug` off.

**Phase B — chrome trim.** Rail geometry, footer, mode label. Pure subtraction; no
behaviour change. *Exit:* rail is tall + slim, no `C:\>_`, no PREP label.

**Phase C — data + fonts.** Imp Lord ID, Grimoire `stage`, drop SB rebuild, font
bumps. *Exit:* Grimoire no longer sits AVAILABLE all fight, the strip shows imp's
**key**, sequence text is legible.

**Phase D — the sequence rework.** D1 arbitration, then D2 burn-window persistence,
then the SpecBurst content. *Exit:* opener survives an immediate in-combat cast (no
flip to burst); pressing Tyrant in a window keeps the pane walking HoG HoG → cores →
Implosion for ~18 s; the burst strip shows the imp step.

Then luaparser-check, bump `.toc`, `wowkb.addon release cdmp`, `/reload`, re-verify.

## Verification (in-game, at a target dummy)

1. **Bleed (A):** 1–2 icons carry a coloured bleed at a glance; Implosion reads **cyan
   "ready"** when off cooldown; `hud debug` off shows no reason text and the board is
   still readable; `hud debug` on restores full rows.
2. **Chrome (B):** rail spans the full stack, is visibly narrower, no `C:\>_`, no PREP
   label; mode still legible via tint.
3. **Data (C):** Grimoire is **not** permanently AVAILABLE; in the sequence strip the
   imp step shows a **keybind**, not "Imp Lord"; the opener has no trailing SB step.
4. **Opener not clobbered (D1):** opener on, enter combat and cast immediately → the
   pane **stays the opener** (does not flip to burst); it drains as you play.
5. **Burn persists (D2):** reach a Tyrant window → pane arms, stage Dreadstalkers/Imp
   Lord, press Tyrant → the pane **keeps walking** HoG HoG → cores → Implosion for
   ~18 s (does not vanish at the cast); dissolves on drain/clock/combat-end.
6. **`/cdmp hud log`** — the recorder captures it; read the `queue` events for the
   arm/advance/dissolve timing and tune `BURN_WINDOW` off them if needed (not the same
   session — the M3e doctrine).

## Open questions / risks

- **Non-verbose = no row text at all** vs. keeping a bare level word. Chosen: no text
  (the bleed is the signal), keep the `lit` summary. If the bleed under-reads in play,
  the fallback is a one-word level tag, not the full reasons. Decision revisitable off
  the first play-test.
- **`SetGradient` API shape** varies across client generations (`Color` objects vs a
  raw 8-arg form). Wrap in the same defensive `pcall` idiom as the dot's mask/scale
  setters, falling back to a flat solid bleed so a rename degrades to "less pretty",
  never a paint-path throw.
- **Dropping the [X1] mode label** leans mode on colour. Accepted for GENERATE/SPEND/
  PREP (ambient); BURST — the consequential one — still names itself on the pane. If
  mode legibility regresses, restore a compact glyph-only tag, not the full label.
- **`stage`-ing Grimoire** assumes it is always held for Tyrant. True for the paired
  2-min cadence (`diabolist-sequences.md`), but on the off-Tyrant cycle it should still
  be usable — `optional` in the queue + `stage` only biting inside the BURST window
  keeps it from being wrongly suppressed in sustain. Watch the dummy log for a Grimoire
  that reads "stage" when there is no near Tyrant.
- **`BURN_WINDOW` is a new drifting clock** (like `TYRANT_WINDOW`). It only ever holds
  the pane *up* a bit long — the safe direction — and combat-end / drain both cap it.
  Tune off `/cdmp hud log`, never invent a press from it.
- **Audio stays M6.** All juice remains visual; the `HudChrome.lua:920` no-`PlaySound`
  fence and the `onCorrect/onMiss/onComplete` seams are untouched.
