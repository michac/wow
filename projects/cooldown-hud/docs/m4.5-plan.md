## Why now

The addon has grown to ~15 Lua modules and a real scoring/queue engine, and every pass
so far has been verified **only in-game** — expensive, serial, and blind to regressions
in the pure logic. The release flow already runs a **luaparser syntax gate**
(`wowkb.addon release cdmp`), but that only catches *parse* errors. Three rungs above it,
in increasing cost:

1. **luacheck** — cheap, local, catches undefined globals + unused vars + typos.
2. **busted unit tests** — the prize: the pure-logic modules are exhaustively testable
   off-game, so a scoring/queue regression fails a test instead of a play-test.
3. **`/cdmp selftest`** — the only way to assert against the *live* Secret-Value / CDM
   API paths that busted can't reach.

None of it changes what the player sees; all of it makes every future M4.x/M5 pass
cheaper and safer.

## Approach

### T1 — luacheck (static analysis, local/CI)

- Add a **`.luacheckrc`** at the addon repo root with a **WoW-globals std**: the Blizzard
  API surface (`CreateFrame`, `hooksecurefunc`, `GetTime`, `C_NamePlate`, `UnitPower`,
  event/enum constants, …) as read-globals and our `ns` table as read/write. **Seed the
  std from an existing community WoW `.luacheckrc`** (e.g. the widely-used WoW globals
  list) rather than hand-listing the API — hand-listing is how the signal drowns in false
  positives on `CreateFrame` et al.
- Config: `unused = false` is *too* loose; instead allow-list the intentional unused
  (event handler `event, a1..aN` tails) via `unused_args = false` and curate the rest.
- **Wire it into the release flow** as the rung above luaparser: `wowkb.addon release
  cdmp` gains a `luacheck CDMProbe/` step (non-zero exit blocks the cut), mirroring the
  existing luaparser gate. Document the local invocation in the addon `CLAUDE.md`.
- **Curate the config, don't inline-suppress.** Prefer fixing (or std-listing) over
  scattering `-- luacheck: ignore`.

⚠ **LuaLS + WoW API type definitions is explicitly deferred** to its own follow-up — it
is a bigger lift (a types repo, editor config) for a smaller marginal catch than luacheck
gives. Note it in `kb-inbox.md` / here; don't bundle it.

### T2 — busted unit tests (the prize)

- **Layout:** `CDMProbe/tests/` with `spec/*_spec.lua` and a shared **`mock_ns.lua`**.
  Run with `busted` locally; document in the addon `CLAUDE.md`.
- **⚠ The harness is the lift, not the specs — size it honestly.** The target modules are
  frame-*light*, not frame-*free*, and HudScore is pure of frames but NOT pure of the `ns`
  surface. `mock_ns.lua` has to provide **three** things, and under-scoping this is the
  main risk to T2:
  1. **A minimal `CreateFrame` stub** — a table whose `CreateFontString` /
     `CreateAnimationGroup` / `CreateTexture` / `SetPoint` / `SetText` / `Show` / `Hide` /
     … are no-ops returning chainable stubs. **HudQueue is NOT frame-free** — `buildFrame`
     calls `CreateFrame` and `Advance → render → strip:SetText`, so its mechanics can't be
     exercised without this — and **HudNapkin** needs it too for its module-level
     `ev = CreateFrame(...)`. Plus globals: `GetTime` (a settable fake clock — the napkin
     and LATE both need to advance time), `wipe`, `InCombatLockdown`, `C_Timer`.
  2. **The REAL `SpecDemonology.lua`, loaded as-is** — it is pure data + pure lookup
     functions (`ns.Spec`, `SpecInfo`, `SpecColor`, `SpecPole`, `SpecGhost`, `SpecIDs`,
     `SpecProcGlow`, `SHARD_CAP`). Loading the real table (not stubbing it) means the
     specs test HudScore against the SHIPPING data, and the emphasis/proc/cost bits stay
     honest by construction.
  3. **A fake STATE surface for HudScore** — this is the bulk of the mock, because
     `HudScore.For` reads far more than the plan's old one-line list: `ns.HudState`
     (`.override`, `.Mode()`, `.ProjectedShards()`, `.shards`, `.SourcePresent()`,
     `.aoe`), `ns.ShardCost`, `ns.BaseCooldown`, `ns.HudChrome.GetReady`,
     `ns.HudNapkin.Remaining/SourceOf/SOON_LEAD`, `ns.SpellName`, `ns.IsSecret`,
     `InCombatLockdown`. The specs set these to fixtures per case (shards N, ready
     true/false/nil, mode PREP/SPEND/BURST, a core/art proc present, a napkin remaining).
- **The pure-logic modules are the target:**
  - **`HudScore`** — `(readable state) → (level, reasons, judgeReady, emphasis)`. Assert:
    the strictness invariant (a state that should be NEVER never lights ROTATION —
    §0.5.8.2(c) as a test, not a hope); `judgeable = false` caps at AVAILABLE + sets
    `judgeReady` when otherwise up (Implosion); the builder priority (IB > DB > SB, the
    M4.3 `betterBuilder` guard); **`emphasis = "burst"` reaches the output for Tyrant and
    nil for everything else** (M4.4-A3, now shipped); the BURST/PREP build-to-cap and
    stage-for-Tyrant prunes; cost-gated abilities gate on `shards >= cost`.
  - **`HudQueue`** — drain/skip/count/prime mechanics (needs the frame stub above): prime
    → un-prime on first match; **the C1 invariant** (M4.4, now shipped — an un-primed
    queue always brightens the current step even with `ready=false`, and the dim-until-
    ready gate only bites while primed); drop-through on out-of-order presses; optional-
    step skip when its ability is on cooldown (`stepSkipped`); the `+N more` overflow.
    (`playCastStart` / `playPop` are animation-only — assert they don't error, not their
    visuals.)
  - **`HudNapkin`** — the countdown + the three honesty rules against the fake clock: an
    observed `Available` edge (`Clear`) retires the estimate (ground truth wins); an
    expired estimate reads "should be up, unconfirmed" (`Remaining == 0`) and never
    promotes a dot; a `read` seed overwrites a `cast` estimate (precedence); readability
    is reported, not assumed.
  - **`HudBurst.isBurstStep`** — a small pure-logic add (M4.4-E): it reads the real
    `ns.SpecBurst` + `SpecInfo(...).burstAlign`, so a one-liner spec confirms the float is
    scoped to burst steps (Tyrant/Dreadstalkers/HoG/DB/Implosion in, Shadowfury out).
  - **Frame-heavy, so OUT of busted (T3 + the human pass):** **`HudChrome`** (textures,
    gradients, masks), **`HudFloat`** (pooled FontStrings + nameplate anchor), and the
    cast-start shimmer's *visuals*. Not worth mocking pixels.
- **Fixture discipline:** the state shapes the specs feed become the shared fixtures T3's
  `/cdmp selftest` re-asserts live, so a case proven off-game can be re-checked on-game
  from the same inputs.

- A new `/cdmp selftest` slash command (registered in `Core.lua`, logic in a new
  `HudSelftest.lua`) that runs assertions against **known states** and prints pass/fail
  to chat (and folds a summary into `/cdmp probe`).
- **Scope:** the paths busted can't reach — live **Secret-Value** reads (a secret gate
  caps at AVAILABLE), **override** resolution (a transformed button scores as what it
  became), **cast-phase readability** (does `UNIT_SPELLCAST_SUCCEEDED` carry a non-secret
  spellID *here*), and the registry binding. It **feeds** synthetic-but-live states
  through the same `HudScore`/`HudQueue` entry points the busted specs use, so the two
  agree by construction.
- **Value:** it is the cheap in-game confidence check that replaces "eyeball the whole
  board" for the parts that are logic, leaving the human pass for the parts that are
  *feel*.

## Build order

1. **T1 luacheck** first — it is the cheapest, finds real bugs immediately (including a
   pass over the just-shipped M4.4 code), and its `.luacheckrc` std is reused nowhere else
   so it can't wait on the tests.
2. **T2 busted** — **`mock_ns.lua` is the critical path** (the harness above), so build it
   first against the EASIEST consumer (`HudNapkin` — smallest surface, proves the frame
   stub + fake clock), then `HudScore` (highest value, exercises the full state surface),
   then `HudQueue` (proves the frame stub carries render), then the `HudBurst.isBurstStep`
   one-liner. Prove the tests bite by breaking a rule and watching the expected spec fail.
3. **T3 `/cdmp selftest`** — built on the same fixtures, ships in a normal addon release
   (the next `.toc` bump, → v0.25.0).

T1 + T2 need **no release** (they run against the source tree). T3 rides a normal `.toc`
bump + `wowkb.addon release cdmp` since it adds a slash command.

## Verification

- `luacheck CDMProbe/` exits clean (or curated-clean with justified allow-lists).
- `busted` runs green locally; a deliberately-broken scoring rule fails the expected spec
  (prove the tests bite).
- `/cdmp selftest` in-game prints all-pass; a known secret-gated ability asserts capped-at-AVAILABLE.

## Open questions / risks

- **The mock surface is the real risk, not the specs.** `HudScore.For` reads a wide `ns`
  surface (HudState/ShardCost/GetReady/napkin — see T2); if `mock_ns.lua` is under-built
  the specs pass against a fiction. Mitigation: load the REAL `SpecDemonology` (data +
  lookups), mock only the STATE, and cross-check with T3 (which runs the same entry points
  against live state) so a mock that drifts from the client shows up.
- **`.luacheckrc` false-positive tax.** The WoW-globals std must be right or the signal
  drowns. Seed from a maintained community list; curate, don't inline-suppress.
- **busted's reach stops at frames.** Textures/gradients/masks/nameplate anchors
  (`HudChrome`, `HudFloat`, the shimmer visuals) stay an in-game concern (T3 + the human
  pass). The frame *stub* carries HudQueue/HudNapkin's structural calls; it does NOT try
  to simulate rendering — don't chase pixel coverage in busted.
- **Fixture drift.** T2 and T3 share state shapes on purpose; if they drift, the in-game
  check stops corroborating the off-game one. Keep the fixtures in one place both read.
- **LuaLS deferred, not dropped.** Park the type-definitions idea; revive only if
  luacheck's catch rate proves insufficient.
