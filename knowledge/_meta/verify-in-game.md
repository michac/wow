# In-Game Verification Checklist

Open KB claims that can only be confirmed **while logged into the game** —
"confirm in-game", "verify %s at the obelisk", "which currency does this vendor
use", and the like. Scattered across ~40 files as free-text TODOs they rot
silently, because resolving them needs you in-game but the KB workflow is
desk-side, so they never surface at the right moment. This file is the one place
they collect, and `/sync-characters` surfaces it when you're actually online.

Sibling of `../planning/todo.md` (which is a *per-reset activity* ledger, planner-
populated, cleared weekly). This list is *KB-integrity* — claims to confirm — and
persists until each item is resolved. Both share the eventual endgame of a
PlannerState in-game checklist UI.

## The convention

Tag the claim's line, wherever it lives, with the literal marker
`@verify-ingame` (bare — a mention inside `backticks` is treated as prose and
ignored). Put it on the most descriptive line of the item (the first line of a
multi-line bullet). Example:

```markdown
- [ ] Confirm the depleted/over-time M+ drop count in-game (Encomplete) @verify-ingame
```

The list below is **generated** from those markers by `wowkb.gen_verify` — the
same pattern as `candidates.json` ← `activities/*.md`. **Do not hand-edit the
generated block**; edit the source tag instead.

```bash
uv run python -m wowkb.gen_verify            # rewrite the block below
uv run python -m wowkb.gen_verify --check     # exit 1 if stale (CI guard)
uv run python -m wowkb.gen_verify --print     # stdout only
```

## Resolution protocol

When you confirm an item in-game:

1. Go to the source line (the `path:line` on each item is the anchor).
2. **Edit the claim**: apply what you learned, update `confidence:` if it moved,
   and stamp the date.
3. **Remove the `@verify-ingame` marker** from that line.
4. Re-run `wowkb.gen_verify` — the item drops off this list automatically.

Adding a new one? Just drop the marker on the claim and re-run the generator.
